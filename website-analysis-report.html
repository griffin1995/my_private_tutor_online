Executive Technical Glossary

Comprehensive explanation of distinguished technical terms and advanced technologies employed

⚛️ Frontend Framework Terms

  • Next.js 15: Latest React framework providing server-side rendering, routing, and performance optimisations.
  • React 19: Latest version of React library for building user interfaces with component-based architecture.
  • TypeScript 5.3+: Strongly-typed JavaScript superset providing compile-time error checking and better IDE support.
  • App Router: Next.js file-system based routing with support for layouts, nested routes, and loading states.
  • Server Components: React components that render on the server, reducing client-side JavaScript bundle size.
  • Suspense: React feature allowing components to wait for asynchronous operations with loading states.
  • Streaming: Technique that sends HTML to browser incrementally for faster perceived performance.

🎨 Design & Styling Terms

  • Tailwind CSS 4.x: Utility-first CSS framework providing low-level utility classes for custom designs.
  • Design Tokens: Named entities storing visual design attributes like colours, spacing, and typography.
  • Custom Properties: CSS variables that can be reused throughout stylesheets and updated dynamically.
  • Responsive Design: Web design approach ensuring optimal viewing experience across all device sizes and orientations.
  • Mobile-First: Design strategy starting with mobile layouts then progressively enhancing for larger screens.
  • Dark Mode: Alternative colour scheme using darker backgrounds to reduce eye strain and battery usage.
  • Google Fonts: Web font service providing optimized font loading and rendering performance.

⚡ Performance Terms

  • Core Web Vitals: Google's metrics measuring real-world user experience: LCP, FID/INP, and CLS.
  • LCP (Largest Contentful Paint): Time until the largest content element becomes visible to users.
  • FID/INP (First Input Delay/Interaction to Next Paint): Time between user interaction and browser response.
  • CLS (Cumulative Layout Shift): Measure of visual stability during page loading.
  • Bundle Optimization: Techniques reducing JavaScript file sizes through code splitting and tree shaking.
  • Tree Shaking: Process removing unused code from final JavaScript bundles to optimise performance.
  • Code Splitting: Breaking application into smaller chunks loaded only when needed.
  • Lighthouse: Google's automated tool auditing web page performance, accessibility, and SEO compliance.

♿ Accessibility Terms

  • WCAG 2.1 AA: Web Content Accessibility Guidelines level AA compliance ensuring content is accessible to users with disabilities.
  • ARIA Labels: Attributes providing screen readers with additional context about elements and their purpose.
  • Semantic HTML: Using HTML elements according to their intended meaning rather than visual appearance.
  • Focus Management: Controlling keyboard focus movement to ensure logical navigation flow and accessibility.
  • Screen Reader: Assistive technology converting text and interface elements to speech or braille.
  • Keyboard Navigation: Ability to navigate and interact with web content using only keyboard input.
  • axe-core: Accessibility testing engine providing automated accessibility violation detection.

🛡️ Security Terms

  • JWT (JSON Web Tokens): Secure method for transmitting information between parties as JSON objects.
  • CSRF Protection: Security measures preventing cross-site request forgery attacks on authenticated users.
  • HTTP-only Cookies: Cookies that cannot be accessed by client-side JavaScript, improving security.
  • SSL/TLS: Cryptographic protocols securing communication between web browsers and servers.
  • Security Headers: HTTP response headers providing additional security against common attack vectors.
  • Input Validation: Process of checking user input for malicious content before processing.
  • Zod: TypeScript schema validation library ensuring data integrity and type safety.

🧠 Content Management Terms

  • CMS (Content Management System): System allowing non-technical users to manage website content without coding.
  • Headless CMS: Content management system providing content via API without dictating frontend presentation.
  • JSON Schema: Vocabulary for validating structure of JSON data and ensuring consistency.
  • Type Safety: Programming language feature preventing type-related errors at compile time.
  • Content Validation: Process ensuring content meets specified format and quality requirements for consistency.
  • Version Control: System tracking changes to content and code over time with history.
  • React Cache: React feature for memoizing expensive computations and API calls.

🔧 Development Tools Terms

  • ESLint: Tool analyzing JavaScript code for potential errors and enforcing coding standards.
  • Prettier: Code formatter ensuring consistent code style across the entire project.
  • Husky: Tool enabling Git hooks to run scripts before commits and pushes.
  • Jest: JavaScript testing framework providing unit testing capabilities with mocking support.
  • Playwright: Cross-browser automation library for end-to-end testing of web applications.
  • Bundle Analyzer: Tool visualising webpack bundle contents to identify optimisation opportunities.
  • Webpack: Module bundler transforming and combining JavaScript modules and assets.

🔄 DevOps & Deployment Terms

  • CI/CD: Continuous Integration/Continuous Deployment automating code testing and deployment processes.
  • GitHub Actions: Platform automating software workflows including testing and deployment.
  • Vercel: Cloud platform optimized for frontend frameworks with global CDN distribution.
  • CDN (Content Delivery Network): Network of servers delivering web content based on user geographic location.
  • Edge Computing: Processing data closer to users rather than centralised servers for better performance.
  • Container: Lightweight, portable package containing application and its dependencies.
  • Load Balancing: Distributing network traffic across multiple servers for reliability and performance.

📊 Analytics & Monitoring Terms

  • Real User Monitoring (RUM): Performance monitoring using data from actual users browsing the website.
  • Synthetic Monitoring: Performance testing using automated scripts simulating user interactions.
  • Error Boundary: React component catching JavaScript errors anywhere in component tree.
  • Performance Budget: Set limits on performance metrics preventing performance regressions.
  • Observability: Ability to understand system internal state by examining outputs and logs.
  • Telemetry: Automated collection and transmission of data from remote sources.
  • Business Intelligence: Technologies and strategies for analyzing business information and data.

🔍 SEO & Marketing Terms

  • SEO (Search Engine Optimisation): Practice of increasing website visibility in search engine results.
  • Meta Tags: HTML elements providing metadata about web pages to search engines.
  • Open Graph: Protocol enabling web pages to become rich objects in social media platforms.
  • Structured Data: Standardized format providing information about page content to search engines.
  • Sitemap: File listing all website pages to help search engines discover content.
  • Robots.txt: File instructing web crawlers which pages they can access on the website.
  • Canonical URL: Preferred version of web page when multiple URLs have similar content.

🏗️ Architecture Terms

  • Component Architecture: Design approach building user interfaces from reusable, modular components.
  • Modular Design: Approach dividing system into separate, interchangeable components with specific functionality.
  • API-First: Development approach designing APIs before implementing user interfaces or applications.
  • Scalable Architecture: System design supporting increased workload without performance degradation.
  • Microservices: Architecture pattern decomposing applications into smaller, independently deployable services for scalability.
  • State Management: Handling application state across components and user interactions.
  • Dependency Injection: Design pattern providing dependencies to components rather than creating them internally.

🎯 UI/UX Terms

  • Progressive Web App (PWA): Web application using modern capabilities to provide app-like experience.
  • Micro-interactions: Small animated responses to user actions improving engagement and usability.
  • Design System: Collection of reusable components and guidelines ensuring consistent design.
  • User Experience (UX): Overall experience person has when interacting with product or service.
  • User Interface (UI): Visual elements and controls users interact with in software application.
  • Responsive Breakpoints: Specific screen widths where website layout adapts for different devices.
  • Touch Optimisation: Designing interfaces specifically for touch-based interactions on mobile devices.

⚖️ Legal & Compliance Terms

  • GDPR: General Data Protection Regulation ensuring data privacy rights for EU citizens.
  • Cookie Consent: Legal requirement obtaining user permission before storing cookies on their device.
  • Privacy Policy: Legal document explaining how personal data is collected, used, and protected.
  • Terms of Service: Legal agreement defining rules and guidelines for using website services.
  • Data Protection: Practices and policies ensuring personal information security and privacy.
  • Cookie Policy: Document explaining types of cookies used and user control options.
  • Legal Compliance: Meeting all applicable laws, regulations, and industry standards.
  • User Rights: Legal entitlements users have regarding their personal data and privacy.
